# Skill: mw-config

Teaches Claude Code how to interact with projects using `@merrywhether/config`.

## Overview

Projects configured with `@merrywhether/config` have a single source of truth:
`mw.config.ts`. All tool configuration (ESLint, TypeScript, Prettier, Renovate,
etc.) flows from this file through projen synthesis.

## Core rules

1. **Never edit generated files.** Any file with a
   `Generated by mw-config. DO NOT EDIT` header is overwritten on the next
   synth run. Editing them is futile.
2. **Edit `mw.config.ts` only.** This is the sole file a human or LLM agent
   should modify for configuration changes.
3. **Run `pnpm pj` after editing.** This re-synthesizes all generated files
   from the new config.
4. **Verify with `mw-config --describe`.** Prints the current parsed config as
   JSON — pipe it into your prompt to understand what is active.

## File roles

| File | Editable | Purpose |
|------|----------|---------|
| `mw.config.ts` | ✓ YES | Single source of truth |
| `.projenrc.ts` | no | Generated thin wrapper that reads `mw.config.ts` |
| `eslint.config.ts` | no | Generated from `mw.config.ts` |
| `prettier.config.ts` | no | Generated from `mw.config.ts` |
| `tsconfig.json` | no | Generated from `mw.config.ts` |
| `renovate.json5` | no | Generated from `mw.config.ts` |
| `.mise.toml` | no | Generated from `mw.config.ts` |
| `pnpm-workspace.yaml` | no | Generated from `mw.config.ts` |

## Schema reference

```typescript
import { defineConfig } from '@merrywhether/config';

export default defineConfig({
  // Optional: override the built-in preset
  // preset: myCustomPreset,

  project: {
    linting: {
      tool: 'eslint',                   // 'eslint' | 'biome' | 'oxlint' | null
      presets: ['base', 'react'],       // stackable: base < typescript < react|solid|vue
      customConfig: false,              // true enables eslint.config augmentation block
      extraIgnores: false,              // true enables eslint.extraIgnores array below
    },
    formatting: {
      tool: 'prettier',                 // 'prettier' | 'biome' | 'oxfmt' | null
      customConfig: false,
    },
    depManagement: {
      tool: 'renovate',                 // 'renovate' | 'dependabot' | null
      preset: 'app',                    // 'app' | 'package'
    },
    typecheck: {
      tool: 'typescript',              // 'typescript' | null
      presets: ['base'],               // stackable: base, solid, styled, vue
      include: ['src'],                // paths to include in tsconfig
    },
    copyright: 'Author Name',
    license: 'MIT',                    // SPDX string | false
    mise: 'lts',                       // version string | false
    ignorePatterns: false,             // true enables ignorePatternsList array below
  },

  // Hand-edited arrays — present when corresponding boolean is true
  ignorePatternsList: ['.next/', 'out/'],
  eslint: {
    extraIgnores: ['**/*.d.ts'],
    config: [],                        // additional ESLint flat config entries
  },
  prettier: {
    config: {},                        // additional Prettier options
  },
  pnpm: {
    onlyBuiltDependencies: [],         // packages allowed to run install scripts
  },
});
```

## ESLint preset hierarchy

Presets are cumulative — each extends the previous:

```
base → typescript → react
                 → solid
                 → vue
```

Selecting `['react']` is equivalent to `['base', 'typescript', 'react']`.
The projen component automatically picks the most comprehensive preset.

## TypeScript preset stacking

Multiple TS presets can be combined:
- `base` — core strict settings (always include this)
- `solid` — Solid-specific additions
- `styled` — styled-components additions
- `vue` — `@vue/typescript-plugin`, includes `**/*.vue`

## Common tasks

### Add React support
```typescript
linting: { tool: 'eslint', presets: ['react'] },
typecheck: { tool: 'typescript', presets: ['base'] },
```
Then run `pnpm pj`.

### Enable Renovate
```typescript
depManagement: { tool: 'renovate', preset: 'app' },
```
Then run `pnpm pj`.

### Add custom ESLint rules
Set `customConfig: true` in linting, then add to the `eslint.config` array:
```typescript
linting: { tool: 'eslint', presets: ['typescript'], customConfig: true },
eslint: {
  config: [{ rules: { 'no-console': 'error' } }],
},
```

### Change mise Node version
```typescript
mise: '24.15.0',
```
Or set to `false` to disable mise config generation.

## Workflow

```sh
# 1. Inspect current config
mw-config --describe

# 2. Edit mw.config.ts
$EDITOR mw.config.ts

# 3. Re-synthesize
pnpm pj

# 4. Verify no unintended changes
git diff

# 5. Commit everything
git add -A && git commit
```
